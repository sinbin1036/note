## 1번 데이터 통신중에 메일 송수신하는 과정순서

**1. 이메일 송신 (송신자 → 메일 서버)** 송신자가 컴퓨터나 스마트폰 등을 이용해 이메일을 작성하고 전송합니다. 이때 데이터는 네트워크를 통해 메일 서버로 전달됩니다. 기술적으로는 응용 계층의 **SMTP(Simple Mail Transfer Protocol)** 프로토콜(포트 25번)을 사용하여 메일을 전송합니다,.

**2. 메시지 저장 (메일 서버)** 송신자가 보낸 메시지는 메일 서버에 도착하여 저장됩니다. 기업 네트워크의 경우, 메일 서버는 외부와 내부망의 중간 지점인 **DMZ(DeMilitarized Zone)** 영역에 위치하여 외부 사용자와의 메일 송수신을 담당하기도 합니다.

**3. 수신 알림 (메일 서버 → 수신자)** 메일 서버는 수신자에게 새로운 메시지가 도착했음을 알립니다.

**4. 이메일 수신 및 읽기 (수신자 ← 메일 서버)** 수신자는 서버에 접속하여 메시지를 검색하고 이메일을 읽습니다. 이때는 주로 **POP3(Post Office Protocol version 3)** 프로토콜(포트 110번)이나 IMAP 등을 사용하여 서버에 저장된 메일을 수신자의 기기로 가져옵니다,.

이 과정에서 데이터는 패킷(Packet)이라는 작은 단위로 쪼개져 전송되며, 각 계층(전송 계층의 TCP, 네트워크 계층의 IP 등)을 거치며 헤더가 붙는 캡슐화 과정을 통해 전달됩니다.

## 2번 영문 텍스트 메신저에서 임의의 영문 텍스트 메시지를 전송한다고 가정하라. 이 텍스트가 네트워크 상에서 패킷으로 분할되어 전송되는 과정을 설명하고, 각 패킷의 비트 단위 데이터에 대해 **홀수 패리티(odd parity)**와 짝수 패리티(even parity) 방식을 각각 적용하여 패리티 비트를 생성·부착하고 송신·수신 측에서 오류를 검출하는 전체 절차를 상세히 서술)

1. 텍스트 메시지의 패킷 분할 및 전송 과정

메신저에서 영문 텍스트(예: "Jin")를 전송할 때, 데이터는 다음과 같은 과정을 거쳐 네트워크로 전송됩니다.

• **데이터의 디지털 변환:** 컴퓨터는 모든 데이터를 0과 1의 비트(bit)로 처리합니다. 전송하려는 문자(알파벳 등)는 아스키코드(ASCII code)와 같은 약속된 코드표에 따라 이진수 비트열로 변환됩니다.

    ◦ 예) J: 1001010, i: 1101001, n: 1101110

• **패킷 분할 (Packet Segmentation):** 용량이 큰 데이터는 한 번에 보내지 않고 **패킷(Packet)**이라는 작은 단위로 쪼개어 전송합니다. 각 패킷은 데이터의 본문(페이로드)과 헤더로 구성됩니다. 헤더에는 데이터의 순서를 알 수 있는 일련번호, 송수신지 정보 등이 포함됩니다.

• **랜덤 전송 및 재조립:** 분할된 패킷들은 네트워크 상황에 따라 서로 다른 경로로 전송될 수 있으며, 수신 측에 순서 없이 랜덤하게 도착할 수 있습니다. 수신 측 컴퓨터는 헤더에 붙은 번호를 확인하여 패킷을 원래 순서대로 재조립함으로써 원본 메시지("Jin")를 복원합니다.

--------------------------------------------------------------------------------

2. 패리티 비트를 이용한 오류 검출 전체 절차

패리티 비트 검사(Parity Bit Check)는 전송하는 데이터(7비트 또는 8비트) 끝에 1비트의 검사 비트(패리티 비트)를 추가하여 전송 과정에서 발생한 비트 오류를 검출하는 방식입니다.

**A. 홀수 패리티 (Odd Parity) 방식**

전체 비트(데이터+패리티)에서 **1의 개수가 홀수**가 되도록 패리티 비트를 생성합니다.

**① 송신 측 (패리티 생성 및 부착):**

• 데이터의 1의 개수를 셉니다.

• **1의 개수가 짝수인 경우:** 패리티 비트를 **1**로 설정하여 전체 1의 개수를 홀수로 만듭니다.

• **1의 개수가 홀수인 경우:** 패리티 비트를 **0**으로 설정하여 전체 1의 개수를 홀수로 유지합니다.

    ◦ _사례 (문자 'Jin' 전송 시)_:

        ▪ **J (1001010):** 1이 3개(홀수) → 패리티 **0** 부착 → 전송 데이터: `0`1001010

        ▪ **i (1101001):** 1이 4개(짝수) → 패리티 **1** 부착 → 전송 데이터: `1`1101001

        ▪ **n (1101110):** 1이 5개(홀수) → 패리티 **0** 부착 → 전송 데이터: `0`1101110

**② 수신 측 (오류 검출):**

• 수신된 패킷(데이터+패리티) 내의 1의 총 개수를 셉니다.

• **1의 개수가 홀수이면:** 오류가 없는 것으로 간주하고 데이터를 수락(Accept)합니다.

• **1의 개수가 짝수이면:** 전송 중 비트가 깨졌다고 판단하여 오류를 검출(Reject)하고 재전송을 요청합니다.

--------------------------------------------------------------------------------

**B. 짝수 패리티 (Even Parity) 방식**

전체 비트(데이터+패리티)에서 **1의 개수가 짝수**가 되도록 패리티 비트를 생성합니다.

**① 송신 측 (패리티 생성 및 부착):**

• 데이터의 1의 개수를 셉니다.

• **1의 개수가 홀수인 경우:** 패리티 비트를 **1**로 설정하여 전체 1의 개수를 짝수로 만듭니다.

• **1의 개수가 짝수인 경우:** 패리티 비트를 **0**으로 설정하여 전체 1의 개수를 짝수로 유지합니다.

    ◦ _사례 (문자 'Jin' 전송 시 응용)_:

        ▪ **J (1001010):** 1이 3개(홀수) → 패리티 **1** 부착 → 전송 데이터: `1`1001010

        ▪ **i (1101001):** 1이 4개(짝수) → 패리티 **0** 부착 → 전송 데이터: `0`1101001

**② 수신 측 (오류 검출):**

• 수신된 패킷 내의 1의 총 개수를 셉니다.

• **1의 개수가 짝수이면:** 오류가 없는 것으로 간주합니다(예: 0 XOR 연산 결과가 0).

• **1의 개수가 홀수이면:** 오류가 발생한 것으로 판단합니다

## 3번 네트워크 명령어 관련된 문제 출제 (이 명령어가 무엇인지? 1줄 정도 서술할수 있도록 출제함)

1. 네트워크 연결 상태 및 진단 명령어

• **ping [IP주소 또는 도메인]**

    ◦ 컴퓨터의 네트워크 상태를 점검하거나 진단하는 명령어로, 상대방 컴퓨터(네트워크)와 제대로 연결되었는지 확인합니다.

• **ping 127.0.0.1 (또는 ping localhost)**

    ◦ 자기 자신에게 신호를 보내 컴퓨터에 내장된 LAN 카드에 이상이 없는지(정상 작동 여부)를 점검합니다.

• **tracert [도메인명]**

    ◦ 목적지(특정 웹사이트나 서버)까지 데이터가 이동하는 경로를 추적하여 보여줍니다.

2. IP 구성 및 정보 확인 명령어

• **ipconfig**

    ◦ 현재 컴퓨터의 네트워크 설정 정보(IP 주소, 서브넷 마스크, 기본 게이트웨이 등)를 확인합니다.

• **ipconfig /all**

    ◦ IP 주소 정보뿐만 아니라 물리적 주소인 MAC 주소(Physical Address)를 포함한 모든 상세 네트워크 정보를 확인합니다.

• **ipconfig /release**

    ◦ 현재 할당받은 IP 주소를 해제하여 네트워크 연결을 끊습니다.

• **ipconfig /renew**

    ◦ DHCP 서버로부터 새로운 IP 주소를 다시 할당받아 네트워크에 연결합니다.

3. ARP(주소 변환) 관련 명령어

• **arp -a**

    ◦ ARP 캐시 테이블을 조회하여 현재 저장된 IP 주소와 그에 매핑된 물리적 주소(MAC 주소) 목록을 확인합니다.

• **arp -d**

    ◦ 기존에 저장되어 있는 ARP 캐시 테이블의 내용을 삭제합니다 (관리자 권한 필요).

4. 시스템 정보 확인 명령어

• **hostname**

    ◦ 현재 사용 중인 컴퓨터의 이름(호스트명)을 확인합니다.

• **net user**

    ◦ 현재 컴퓨터에 등록된 사용자 계정 목록을 확인합니다.

## 4번 그림을 2개 보여줘서 허부를 구별하는 문제(다음중 어떤 허부인지 어떤 특징을 가지고 있는지 출제)

1. 더미 허브 (Dummy Hub)

• **작동 방식:** 데이터를 수신지 컴퓨터뿐만 아니라 **연결된 모든 컴퓨터로 보냅니다**. 수신되는 데이터의 의도한 대상을 식별할 수 없기 때문입니다.

• **대역폭 공유:** 네트워크의 전체 대역폭을 연결된 노드(컴퓨터) 수만큼 나누어 사용합니다. 따라서 연결된 컴퓨터 수가 늘어나면 네트워크 속도가 떨어지는 단점이 있습니다.

• **용도:** 네트워크 트래픽 문제로 인해 주로 10대 이하의 소규모 네트워크 구축에 사용되며, 가격이 저렴한 편입니다.

2. 스위칭 허브 (Switching Hub)

• **작동 방식:** 수신지 주소를 읽어 **해당 주소의 단말기로만 데이터를 보내는 스위칭 기능**이 있습니다. 노드들을 점대점으로 접속시킵니다.

• **대역폭 보장:** 더미 허브와 달리 전체 대역폭을 나누어 쓰지 않고, 연결된 컴퓨터 수에 상관없이 **각 컴퓨터에 할당된 대역폭을 모두 제공**합니다 (예: 100Mbps 대역폭이라면 5대를 연결해도 각자 100Mbps 보장).

• **장점:** 네트워크 효율이 더미 허브보다 훨씬 높습니다.

3. 스태커블 허브 (Stackable Hub)

• **특징:** 스택 접속 포트가 있어 허브와 허브 사이를 연결하여 용량을 확장할 수 있습니다. 여러 대의 허브를 쌓아서 연결하면 **하나의 허브 시스템처럼 동작**합니다.

• **장점:** 일반 포트로 여러 대의 더미 허브를 연결할 때 발생하는 전송 속도 차이 문제를 해결할 수 있어, 데이터 전송 속도 차이가 적습니다. 중규모 이상의 네트워크 구성에 사용됩니다.

4. 인텔리전트 허브 (Intelligent Hub)

• **특징:** 신호의 조절과 변경 등 다양한 지능형 기능을 포함하고 있는 허브입니다.

## 5번 네트워크 접속형태에 대해서 기술하는 문제 (ex. 버스형, 트리형)

노드와 링크가 물리적 또는 논리적으로 배치되는 방식을 말하며, 크게 **성형, 버스형, 트리형, 링형, 그물형, 혼합형**으로 분류됩니다. 각 형태의 특징과 장단점은 다음과 같습니다.

1. 성형 (Star Topology)

가장 일반적인 구성 형태로, 중앙에 위치한 허브(Hub)가 모든 노드를 연결하는 방식입니다.

• **장점:**

    ◦ 각 장치가 중앙 허브와 하나의 링크로만 연결되므로 설치 비용이 저렴합니다.

    ◦ 중앙 집중적 구조라 유지보수와 확장이 쉽습니다.

    ◦ 하나의 링크가 끊어져도 해당 링크만 영향을 받고 나머지 네트워크는 안전합니다.

• **단점:**

    ◦ 중앙의 허브(전송제어장치)가 고장 나면 전체 네트워크가 마비됩니다.

    ◦ 통신량이 많으면 전송 지연이 발생할 수 있습니다.

2. 버스형 (Bus Topology)

모든 노드와 장치들이 하나의 긴 중앙 케이블(버스)에 연결된 형태입니다. 케이블의 양 끝에는 신호 반사를 막기 위한 터미네이터(Terminator)가 부착됩니다.

• **장점:**

    ◦ 설치가 간단하고 케이블 비용이 적게 듭니다.

    ◦ 장비 추가가 쉽고, 특정 장비가 고장 나도 전체 네트워크에 영향을 주지 않습니다.

• **단점:**

    ◦ 중앙 케이블에 결함이나 파손이 생기면 네트워크 전체가 중단됩니다.

    ◦ 문제 발생 시 결함 위치를 찾거나 재구성이 어렵습니다.

    ◦ 트래픽(장비 수)이 많아지면 성능이 저하되며, 충돌이 발생할 수 있습니다.

3. 트리형 (Tree Topology)

성형의 변형으로, 중앙의 전송제어장치에 모든 장비를 직접 연결하는 대신 트리 형태의 계층적 구조로 노드들을 연결하는 방식입니다.

• **장점:**

    ◦ 제어가 간단하여 관리나 네트워크 확장이 쉽습니다.

    ◦ 데이터 전송 거리를 늘릴 수 있으며, 여러 컴퓨터를 분리하거나 우선순위를 부여하기 좋습니다.

• **단점:**

    ◦ 중앙 트래픽 집중으로 병목현상이 발생할 수 있습니다.

    ◦ 성형과 마찬가지로 중앙 전송제어장치가 다운되면 전체 네트워크에 장애가 발생합니다.

4. 링형 (Ring Topology)

모든 컴퓨터를 하나의 고리(Ring) 형태로 순차적으로 연결하는 방식입니다. 각 노드는 양옆의 이웃 노드와만 연결됩니다.

• **장점:**

    ◦ 구조가 단순하여 설치와 재구성이 쉽습니다.

    ◦ 성형보다 케이블 비용을 줄일 수 있으며, 장애 발생 시 복구가 빠릅니다.

• **단점:**

    ◦ 링을 제어하는 절차가 복잡합니다.

    ◦ 새 장비를 추가하려면 링을 절단해야 하며, 단순 링형의 경우 한 장치에 결함이 생기면 전체 네트워크를 사용할 수 없습니다.

5. 그물형 (Mesh Topology)

중앙 제어 노드 없이 모든 노드가 상호 간에 전용회선(점대점)으로 연결된 형태입니다.

• **장점:**

    ◦ 전용 링크를 사용하므로 통신량 문제(트래픽체증)를 해결할 수 있습니다.

    ◦ 한 링크가 고장 나도 다른 경로를 이용할 수 있어 신뢰성이 높습니다.

    ◦ 원하는 수신자에게만 데이터를 보내므로 보안에 유리합니다.

• **단점:**

    ◦ 모든 노드를 연결해야 하므로 설치와 재구성이 어렵고 케이블 비용이 많이 듭니다.

    ◦ 설치 공간을 많이 차지합니다.

6. 혼합형 (Hybrid Topology)

실제 대규모 네트워크에서 효율을 높이고 결함 허용 능력을 키우기 위해 여러 접속 형태(서브넷)를 결합하여 사용하는 방식입니다.

## 6번 첫번째 우선전송매체 동축케이블 (유선전송매체 각각에 대해서 장단점 기술)

제공된 강의 자료에 따르면 유선 전송 매체는 크게 **동축 케이블, 꼬임선(Twisted Pair), 광섬유 케이블**로 나뉩니다. 각 매체의 장단점은 다음과 같습니다.

1. 동축 케이블 (Coaxial Cable)

케이블 TV 시스템과 유사한 방식으로, 두 개의 전도체(중앙 와이어, 차폐용 그물망)로 구성됩니다.

• **장점:**

    ◦ 설치가 용이합니다.

    ◦ 꼬임선보다 더 큰 대역폭을 지원하며, 최대 전송 속도가 빠릅니다.

    ◦ 외부 신호와 전자파를 차단하는 능력이 뛰어나 간섭에 강합니다.

• **단점:**

    ◦ 카테고리 3의 UTP 케이블보다 가격이 비쌉니다.

    ◦ 설치 기술에 따라 관리가 어려울 수 있습니다.

2. 꼬임선 (Twisted Pair)

두 가닥의 절연된 구리선을 나선형으로 꼬아서 만든 케이블로, 차폐 여부에 따라 UTP, FTP, STP로 나뉩니다.

• **장점:**

    ◦ 만들기 쉽고 가격이 저렴합니다,.

    ◦ 비교적 안정적입니다.

    ◦ 광섬유 케이블보다 설치가 용이합니다.

• **단점:**

    ◦ 고속 전송에는 적합하지 않습니다.

    ◦ 감쇠 현상 비율이 높습니다.

    ◦ 전기적 신호의 간섭에 민감합니다.

3. 광섬유 케이블 (Optical Fiber Cable)

빛을 이용하여 데이터를 전송하는 매체로 유리나 플라스틱으로 만들어집니다.

• **장점:**

    ◦ 높은 대역폭을 지원하여 고속 전송이 가능합니다.

    ◦ 감쇠 현상 비율이 낮아 장거리 전송에 유리합니다.

    ◦ 빛을 이용하므로 외부의 전자기적 간섭에 매우 강합니다.

• **단점:**

    ◦ 구축 비용이 비쌉니다.

    ◦ 설치가 복잡합니다.

## 7번 IEEE 802.11AC에 대해 설명

1. IEEE 802.11 개요

IEEE(전기전자공학자협회)는 2.4GHz 등의 주파수 대역을 사용하는 무선 네트워크 표준인 **IEEE 802.11**을 제정하였습니다. 이 표준은 무선 주파수나 적외선 전송을 사용하는 무선 LAN(WLAN) 기술을 정의하며, 무선 LAN의 블록을 형성하는 기본 서비스 집합(BSS)과 이를 확장한 확장 서비스 집합(ESS) 등의 서비스를 규정하고 있습니다.

2. IEEE 802.11ac의 특징

IEEE 802.11ac는 초기 규격인 802.11a, b, g보다 발전된 형태로, 현재 802.11n과 함께 널리 사용되는 무선 LAN 규격입니다. 주요 특징은 다음과 같습니다.

• **주파수 대역:** **5GHz 대역**을 사용합니다. 5GHz 대역은 2.4GHz 대역을 사용하는 다른 단말기(블루투스 기기, 무선 전화기 등)와의 전파 간섭이 적고, 더 넓은 주파수 대역을 활용할 수 있다는 장점이 있습니다.

• **전송 속도:** 최대 전송 속도가 **6.9Gbps**에 달합니다. 이는 2.4GHz 대역에서 최대 300Mbps(802.11n 기준)의 속도를 내는 것과 비교했을 때 매우 빠른 기가비트급 속도입니다.

• **활용:** 고속 전송이 가능하여 대용량 데이터 처리에 적합하며, 최신 무선 LAN 환경 구축에 주로 활용됩니다.

요약하자면, **IEEE 802.11ac**는 간섭이 적은 **5GHz 주파수**를 사용하여 **6.9Gbps의 초고속 데이터 전송**을 지원하는 고성능 무선 LAN 표준입니다.

## 8번 컴퓨터 네트워크 영어 풀네임 즉, 강의에 있는 영어 약자 전체 풀용어를 적는 기술 9개문제 (ex. FTP, LAN )

**1. LAN (Local Area Network)**

• **의미:** **근거리 통신망**을 의미합니다. 한 건물이나 학교 내 캠퍼스, 가정처럼 비교적 가까운 지역에 한정된 통신망입니다. 초기에는 10~100Mbps의 이더넷을 주로 사용했으나, 최근에는 기가비트 이더넷이나 무선 LAN 등 고속 전송 기술이 사용됩니다,,.

**2. ISP (Internet Service Provider)**

• **의미:** **인터넷 서비스 제공자**를 말합니다. KT, SK브로드밴드, LGU+와 같이 개인이나 기업에게 인터넷 접속 서비스, 웹 사이트 구축 및 웹 호스팅 서비스 등을 제공하는 사업자를 지칭합니다. 가정이나 회사가 인터넷(WAN)에 연결하기 위해서는 ISP의 회선을 사용해야 합니다,.

**3. DMZ (DeMilitarized Zone)**

• **의미:** **비무장 지대**라는 뜻으로, 외부 네트워크(인터넷)와 내부 네트워크 사이에 위치한 중간 지점(서브넷)입니다. 기업 네트워크에서 외부의 접근을 허용해야 하는 웹 서버, 메일 서버, DNS 서버 등을 이 영역에 두고, 내부망을 보호하기 위해 구성합니다.

**4. NIC (Network Interface Card)**

• **의미:** 흔히 **LAN 카드**라고 부르며, 컴퓨터를 네트워크에 연결하기 위해 컴퓨터 내부에 설치하는 확장 카드입니다. 전송 매체에 접속하는 역할과 데이터의 입출력, 송수신, 프로토콜 처리 기능 등을 담당합니다. 각 NIC에는 고유한 MAC 주소가 부여되어 있습니다.

**5. OSI (Open System Interconnection)**

• **의미:** **개방형 시스템 상호 연결** 모델입니다. 국제표준화기구(ISO)가 서로 다른 컴퓨터나 시스템 간에 원활한 통신을 위해 만든 7계층의 표준화된 통신 규격입니다. 물리 계층부터 응용 계층까지 7단계로 나누어져 있습니다.

**6. CSMA/CD (Carrier Sense Multiple Access with Collision Detection)**

• **의미:** **반송파 감지 다중 접근/충돌 검출** 방식입니다. 이더넷(Ethernet)에서 데이터 충돌을 방지하기 위해 사용하는 매체 접근 제어 방식입니다. 데이터를 보내기 전에 회선이 사용 중인지 확인(CS)하고, 비어 있으면 데이터를 보내되(MA), 전송 중 충돌이 감지(CD)되면 잠시 대기했다가 재전송하는 규칙을 따릅니다,.

**7. ARP (Address Resolution Protocol)**

• **의미:** **주소 변환 프로토콜**입니다. 논리적인 주소인 **IP 주소**를 물리적인 주소인 **MAC 주소**로 변환하는 역할을 합니다. 통신 대상을 찾기 위해 네트워크 전체에 브로드캐스트 방식으로 "이 IP 주소를 가진 컴퓨터의 MAC 주소가 무엇인지"를 요청합니다,.

**8. TCP (Transmission Control Protocol)**

• **의미:** **전송 제어 프로토콜**입니다. 데이터를 주고받기 전에 연결을 설정(3-way handshaking)하는 **연결 지향형 프로토콜**입니다. 데이터의 전송 순서를 보장하고, 오류가 발생하면 재전송을 요청하는 등 신뢰성 있는 데이터 통신을 제공합니다,.

**9. UDP (User Datagram Protocol)**

• **의미:** **사용자 데이터그램 프로토콜**입니다. 연결을 설정하지 않고 데이터를 보내는 **비연결 지향형 프로토콜**입니다. TCP와 달리 흐름 제어나 오류 복구 기능이 없어 신뢰성은 낮지만, 전송 속도가 빠르기 때문에 실시간 스트리밍이나 DNS, DHCP 등에 사용됩니다,.

## 9번 포트 번호 DNS, HTTP, HTTPS 포트번호 관련된 문제 출제 (포트번호 맞추는 문제 : 포트번호 암기 필요)

**1. FTP (File Transfer Protocol)**

• **포트 번호:** **20, 21**

• **설명:** 파일을 전송하는 데 사용되는 프로토콜입니다. 일반적으로 21번 포트는 제어(Control) 연결에 사용되며, 데이터 전송은 20번 포트 또는 다른 포트를 사용할 수 있습니다,,.

**2. Telnet**

• **포트 번호:** **23**

• **설명:** 원격지의 호스트 컴퓨터에 접속하여 호스트 세션(Host Sessions)을 연결하는 데 사용되는 프로토콜입니다,.

**3. SMTP (Simple Mail Transfer Protocol)**

• **포트 번호:** **25**

• **설명:** 이메일(E-Mail)을 전송(송신)할 때 사용하는 프로토콜입니다,.

**4. DNS (Domain Name System)**

• **포트 번호:** **53**

• **설명:** 도메인 이름(예: www.naver.com)을 IP 주소로 변환하거나 그 반대의 작업을 수행하는 서비스입니다. UDP 프로토콜을 주로 사용합니다,.

**5. HTTP (HyperText Transfer Protocol)**

• **포트 번호:** **80**

• **설명:** 웹 브라우저가 웹 서버에 접속하여 웹 페이지(Web Application)의 내용을 요청하고 전송받을 때 사용하는 프로토콜입니다. 별도로 포트 번호를 입력하지 않아도 웹 브라우저는 기본적으로 80번 포트를 사용합니다,,.

**6. POP3 (Post Office Protocol version 3)**

• **포트 번호:** **110**

• **설명:** 메일 서버에 도착한 이메일을 클라이언트(사용자)가 수신하여 읽을 때 사용하는 프로토콜입니다.

**7. SNMP (Simple Network Management Protocol)**

• **포트 번호:** **161, 162**

• **설명:** 네트워크 관리(Network Management)를 위해 사용되는 프로토콜입니다. 네트워크 장비들의 상태를 모니터링하거나 설정을 변경할 때 사용되며, 주로 UDP를 사용합니다,.

**8. HTTPS (HyperText Transfer Protocol Secure)**

• **포트 번호:** **443**

• **설명:** HTTP의 보안 버전으로, 웹 통신 내용을 암호화하여 주고받을 때 사용합니다. 보안이 필요한 웹 사이트 접속 시 사용됩니다,.

## ==10번== 순환중복검사 문제를 3개를 냈는데 20점 문제 (14분 정도)

### 10-1번 순환중복검사(CRC) 과정에서 전송 데이터 다항식과 생성 다항식이 주어졌을 때 P(x), G(x)를 비트로 표현하는 문제

**[예시 문제]** 순환 중복 검사(CRC)를 수행하기 위해 다음과 같은 전송 데이터 다항식 P(x)와 생성 다항식 G(x)가 주어졌습니다. 이를 각각 2진수 비트열로 변환하여 쓰시오.

• **전송 데이터 다항식** P(x)=x7+x5+x3+x2+1

• **생성 다항식** G(x)=x4+x3+x2+1

**[해설 및 정답]** 다항식을 비트로 변환할 때는 각 차수의 계수가 있으면 '1', 없으면 '0'으로 표기합니다.

1. P(x) **변환:**

    ◦ x7 (있음) → 1

    ◦ x6 (없음) → 0

    ◦ x5 (있음) → 1

    ◦ x4 (없음) → 0

    ◦ x3 (있음) → 1

    ◦ x2 (있음) → 1

    ◦ x1 (없음) → 0

    ◦ 1(x0) (있음) → 1

    ◦ **정답** P(x) **비트열:** **10101101**

2. G(x) **변환:**

    ◦ x4 (있음) → 1

    ◦ x3 (있음) → 1

    ◦ x2 (있음) → 1

    ◦ x1 (없음) → 0

    ◦ 1(x0) (있음) → 1

    ◦ **정답** G(x) **비트열:** **11101**

---


### 10-2번 P′(x), P2′(x) 구하는 문제

**[예시 문제]** 위 문제 1번의 조건(P(x):10101101, G(x):11101)을 이용하여 다음을 구하시오. (단, 나눗셈 연산은 Modulo-2 연산을 사용한다.)

1. FCS(오류 검사 코드) 비트 수를 고려하여 시프트 된 데이터 **P′(x)**를 비트로 쓰시오.

2. CRC 계산 과정을 통해 구해진 나머지를 포함한 최종 전송 데이터 **P2′(x)(교재 표기 P′′(x))**를 비트로 쓰시오.

**[해설 및 정답]**

**1.** P′(x) **구하기**

• FCS 비트 수(n)는 생성 다항식 G(x)의 비트 수보다 1개 적습니다.

• G(x)가 5비트(`11101`)이므로, **FCS 비트 수 n=4**입니다.

• P(x)(`10101101`) 뒤에 0을 4개 붙여 왼쪽으로 시프트 합니다.

• **정답** P′(x)**:** **101011010000**

**2.** P2′(x) **구하기**

• **나눗셈 연산:** P′(x)(`101011010000`)를 G(x)(`11101`)로 나눕니다. 이때 XOR 연산(같으면 0, 다르면 1)을 사용합니다.

    ◦ 자료의 계산 과정에 따르면, `101011010000` ÷ `11101`의 **나머지는 `1011`**이 됩니다.

• **최종 데이터 생성:** 원래 데이터 P(x) 뒤에 계산된 나머지(FCS)를 붙입니다. (혹은 P′(x)에 나머지를 더합니다.)

    ◦ `10101101` (데이터) + `1011` (나머지)

• **정답** P2′(x)**:** **101011011011**

---

### 10-3번 수신 측에서 p2′ 다항식 데이터 예시를 보고 오류 유무를 판별하는 문제

**[예시 문제]** 수신 측 컴퓨터가 네트워크를 통해 **`101011011011`**이라는 데이터를 전송받았습니다. 송신 측과 동일한 생성 다항식 **G(x)=11101**을 사용한다고 할 때, 이 데이터에 오류가 있는지 판별하고 그 이유를 서술하시오.

**[해설 및 정답]**

1. **판별 방법:** 수신된 데이터(P2′(x))를 생성 다항식(G(x))으로 Modulo-2 나눗셈(XOR 연산)을 수행합니다.

2. **계산:**

    ◦ 피제수: `101011011011` (수신된 데이터)

    ◦ 제수: `11101` (생성 다항식)

    ◦ 나눗셈을 수행하면 **나머지가 `0`**이 나옵니다.

3. **판정 기준:**

    ◦ 나머지가 **0**이면: **오류 없음 (정상)**

    ◦ 나머지가 **0이 아니면**: **오류 발생**

4. **정답:**

    ◦ **오류 유무:** **오류 없음**

    ◦ **이유:** 수신된 데이터를 생성 다항식으로 나누었을 때 **나머지가 0**이기 때문입니다.

    ◦ **데이터 단위:** 데이터 (Data)

2. 네트워크 통신에서 OSI 7계층 모델이 필요한 이유

강의 자료에서는 OSI 7계층 모델을 나누는 이유와 필요성을 다음과 같이 설명하고 있습니다.

1. **통신 흐름의 파악 및 이해 용이:** 통신이 일어나는 복잡한 과정을 단계별로 파악할 수 있어 전체적인 흐름을 한눈에 알아보기 쉽고 이해하기 쉽습니다.

2. **유지보수 및 문제 해결의 효율성 (모듈화):** 7단계 중 특정한 곳에 이상이 생기면 다른 단계의 장비나 소프트웨어를 건드리지 않고 이상이 생긴 단계만 수정하여 고칠 수 있습니다.

    ◦ _예시:_ 특정 PC에서만 게임 접속이 안 된다면 응용 계층(소프트웨어) 문제, 모든 PC가 안 된다면 물리 계층(회선)이나 네트워크 계층(라우터) 문제로 범위를 좁힐 수 있습니다.

3. **통신 기술 도입과 확장의 용이성:** 통신 기술의 도입과 통신 기능의 확장을 쉽게 하기 위해 프로토콜을 계층으로 나누어 규정합니다.

4. **상호 호환성 제공 (개방형 시스템):** 서로 다른 컴퓨터나 시스템이 네트워크 구조와 상관없이 통신할 수 있도록(개방형) 하기 위해 표준화된 모델이 필요합니다.

## 12번 전이중 통신(Full Duplex)과 반이중 통신(Half Duplex)의 개념을 설명하고,   각각의 대표적인 예를 하나씩 쓰시오

1. 전이중 통신 (Full-Duplex)

• **개념:** 통신 채널에 접속된 두 대의 단말기가 **동시에 데이터를 송수신할 수 있는 통신 방식**입니다. 데이터의 송신과 수신을 동시에 처리하기 위해 통신 채널 두 개를 이용하는 방식입니다.

• **대표적인 예:** **전화기**가 이에 해당합니다. 전화는 말하는 것(송신)과 듣는 것(수신)을 동시에 할 수 있습니다.

   ◦ _네트워크 장비 예시:_ 스위치(Switch)를 이용한 연결이나 컴퓨터 간 직접 LAN 케이블 연결이 전이중 방식에 해당합니다.

2. 반이중 통신 (Half-Duplex)

• **개념:** 양방향으로 데이터 전송이 가능하지만, **동시에는 불가능하고 한 번에 한쪽 방향으로만 전송할 수 있는 방식**입니다. 한쪽이 데이터를 송신하면 상대방은 수신만 할 수 있으며, 양쪽 단말기의 상호 협력에 따라 송수신 방향을 바꿉니다.

• **대표적인 예:** **휴대용 무전기**가 이에 해당합니다. 무전기는 한 사람이 말하는 동안(송신) 다른 사람은 듣기(수신)만 해야 합니다.

   ◦ _네트워크 장비 예시:_ 더미 허브(Dummy Hub)를 이용한 연결이 반이중 방식이며, 동시에 데이터를 보내면 충돌이 발생합니다.

## 14번 다음 보기 중 Class A 공인 IPv4 주소 범위에 해당하는 것은 어느 것인가?

**예상 문제**

**Q. 다음 보기 중 Class A의 '공인(Public)' IPv4 주소 범위에 해당하는 것은 어느 것인가?**

1. `10.10.10.1`

2. `127.0.0.1`

3. `172.16.0.1`

4. `114.200.10.5`

5. `192.168.0.1`

--------------------------------------------------------------------------------

**정답 및 해설**

**정답: 4번 (****114.200.10.5****)**

**[해설]** 강의 자료에 따르면 **Class A**의 전체 범위는 `0.0.0.0` ~ `127.255.255.255`입니다. 하지만 **공인 IP**로 사용 가능한 범위는 사설 IP와 특수 주소를 제외해야 합니다.

• **Class A 공인 IP 범위:** `1.0.0.0` ~ `9.255.255.255` 그리고 `11.0.0.0` ~ `126.255.255.255`

**[오답 분석]**

1. **10.10.10.1**: Class A에 속하지만, **사설 IP 주소**(`10.0.0.0` ~ `10.255.255.255`)이므로 공인 IP가 아닙니다.

2. **127.0.0.1**: Class A 범위(`0~127`)에 들어가지만, 이는 **시스템 루프백(Loopback) 주소**로 예약되어 있어 공인 IP로 사용하지 않습니다.

3. **172.16.0.1**: **Class B**(`128~191`)에 해당하며, 그중에서도 사설 IP 대역(`172.16.0.0` ~ `172.31.255.255`)입니다.

4. **114.200.10.5**: 첫 번째 바이트가 114로, Class A 공인 IP 범위(`11.0.0.0` ~ `126.255.255.255`)에 포함되므로 **정답**입니다.

5. **192.168.0.1**: **Class C**(`192~223`)에 해당하며, 가정이나 소규모 네트워크에서 주로 쓰는 사설 IP 대역(`192.168.0.0` ~ `192.168.255.255`)입니다.

**[시험 대비 암기 포인트]**

• **Class A 사설 IP:** `10.x.x.x` (10으로 시작하면 무조건 사설)

• **루프백 주소:** `127.x.x.x` (가상 주소이므로 공인 IP 아님)

• **0.0.0.0:** 사용하지 않는 주소

### 15-2번 TCP의 6개 코드 비트(SYN, ACK, FIN, RST, PSH, URG)가 각각 무엇을 의미하는지 설명하시오.

**문제 2: 각 코드 비트의 의미**

강의 자료 [표 6-3]에 명시된 각 비트(1비트 크기)의 의미는 다음과 같습니다. 비트 값이 **1**로 설정되면 해당 기능이 활성화됩니다.

2. **ACK (Acknowledgment):** 확인 응답 번호(Acknowledgment Number) 필드가 유효함을 나타내며, 응답 확인 번호를 사용한다는 뜻입니다.

5. **SYN (Synchronize):** 연결을 초기화하기 위해 순서 번호(Sequence Number)를 동기화할 때 사용합니다.

6. **FIN (Finish):** 데이터 송신을 종료할 때 사용합니다.

### 15-3번 TCP 연결 확립 과정에 대해, 코드 비트(플래그) 설정을 기준으로 3-way Handshaking 절차를 서술하시오.

**문제 3: 코드 비트 설정을 기준으로 한 3-way Handshaking 절차**

TCP 연결 확립 과정(3-way Handshaking)은 데이터를 전송하기 전에 신뢰성 있는 연결 통로를 만들기 위한 과정입니다. 이 과정에서 **SYN**과 **ACK** 비트가 핵심적으로 사용됩니다.

**1단계: 연결 확립 요청 (송신 측 → 수신 측)**

• **동작:** 송신 측 컴퓨터는 수신 측 컴퓨터에게 연결 확립 허가를 받기 위해 요청을 보냅니다.

• **코드 비트 설정:** **SYN 비트를 1로 설정**하여 보냅니다 (SYN=1, ACK=0).

    ◦ _(참고: 이때 순서 번호는 랜덤한 초기값으로 설정됩니다.)_

**2단계: 연결 확립 응답 + 연결 확립 요청 (수신 측 → 송신 측)**

• **동작:** 수신 측은 송신 측의 요청을 허가한다는 응답(ACK)을 보내면서, 동시에 자신도 송신 측에게 데이터 전송 허가를 받기 위한 연결 요청(SYN)을 보냅니다.

• **코드 비트 설정:** **SYN 비트와 ACK 비트를 모두 1로 설정**하여 보냅니다 (SYN=1, ACK=1).

**3단계: 연결 확립 응답 (송신 측 → 수신 측)**

• **동작:** 송신 측은 수신 측의 연결 요청을 받았음을 알리는 허가 응답(ACK)을 보냅니다.

• **코드 비트 설정:** **ACK 비트를 1로 설정**하여 보냅니다 (SYN=0, ACK=1).

이 3단계가 완료되면 양쪽 컴퓨터 간의 논리적인 연결이 성립되어 데이터를 전송할 준비가 끝납니다.
